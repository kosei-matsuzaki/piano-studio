# 🎹 Piano Studio — Keyboard Piano & Chord Trainer

PCのキーボードでピアノを演奏でき、音楽理論を学べて、AIが作曲までするWebアプリです。
ビルド不要・インストール不要。`index.html` をブラウザで開くだけで動きます。

**🚀 デモ（ブラウザでそのまま試せます）: https://kosei-matsuzaki.github.io/piano-studio/**

**作品概要・スクリーンショット・技術解説は [docs/PORTFOLIO.md](docs/PORTFOLIO.md) を参照してください。**

## 起動方法

1. このフォルダの `index.html` をブラウザ（Chrome / Edge 推奨）で開く
2. 何かキーを押す or 鍵盤をクリックすると音声が有効になります

> ブラウザの仕様で「最初のユーザー操作」まで音は鳴りません（自動再生防止のため）。

ローカルサーバーで開きたい場合（任意）:

```bash
# このフォルダで
python -m http.server 8000
# → http://localhost:8000 を開く
```

## モード

| モード | 内容 |
|--------|------|
| **🎵 自由演奏** | 自由に弾ける。押している音と推定コード名を表示。保存した楽譜を参考表示も可能。 |
| **📚 コード学習** | 音楽理論を**楽譜（五線譜）と画面下部のピアノ・リズムグリッド**で視覚的に学ぶ、スライド形式の**全52パート・4章立て**（準備編／メロディ編／コード編／リズム編）カリキュラム。 |
| **🤖 AI作曲** | 自作の学習済みニューラルネット（Transformer）がコード進行・メロディ・強弱を**自動生成**。キー・伴奏・テンポを選んで生成→再生。生成曲は楽譜エディタへ取り込み可能。 |
| **📝 楽譜エディタ** | AI作曲の結果をインポートして編集したり、ゼロから楽譜を作成。ブラウザ（localStorage）に保存でき、自由演奏の参考楽譜としても使える。 |

## キーボード配置（一段鍵盤）

白鍵 = A段、黒鍵 = Q段に割り当てた約2オクターブの鍵盤です（`]` キーまで対応）。
白鍵・黒鍵の並びは**選んだキーに応じて実際のピアノと同じ配置**になります（キーC のとき）:

```
黒鍵(Q段):  W E   T Y U   O P   [
白鍵(A段):  A S D F G H J K L ; : ]
            ド レ ミ ファ ソ ラ シ ド レ ミ ファ ソ
```

各鍵盤にはPCキーと現在の音名（例: C4）が表示されます。

### キー（調）変更 ＝ 鍵盤をスライド

「キー（左端の音）」セレクトで C〜B を選ぶと、**実際のピアノを横にスライドさせて、選んだ音が一番左に来る**ように鍵盤を組み直します（黒鍵の位置も本物どおりに変わります）。

- 例：D を選ぶと、一番左の白鍵が **レ（D）** になり、そこから D・D♯・E・F・F♯… と実際のピアノ配置で並びます。
- 白鍵キー（A S D …）・黒鍵キー（Q段）も、その配置に合わせて自動で割り当て直されます。
- **カリキュラムのコード名・度数・メロディも選んだ調に合わせて自動調整**されます（例：レッスンの「Cメジャー」はキーDでは「D」として表示・出題。同じ指の形＝同じキー位置で弾けます）。

> ※ コード進行の I・IV・V などは、どのキーでも同じキー位置（指の形）で弾けるようになります。

### オクターブ変更

- 画面の「◀ オクターブ下げ / オクターブ上げ ▶」ボタン
- キーボードの **↑ / ↓** キー

で鍵盤全体の音域を上下にシフトできます（範囲: オクターブ1〜7）。

## AI作曲（自作の学習型ニューラルネット）

**曲データから学習した小型ニューラルネット**がコード進行・メロディ・強弱を生成します。手書きのルールではなく、学習データに現れた傾向を学びます。

- **3ブロック構成（すべて自作・学習型）**：① コード進行 Transformer → ② コード条件つきメロディ Transformer（16分音符まで。強拍・ロング音にはハモリも付与）→ ③ 強弱曲線 LSTM（実演奏ベロシティから学習）。伴奏は生成コードから濁らない声部を組み立てるルールベースのアレンジャー。
- **豊かなコード（テンション・分数）**：データセットの実表記を正確に解析し、**9th/11th/13th・add9・aug・dim・sus・m7♭5・分数（スラッシュ）コード**を保持して学習。「多様性＝大胆」ほど出やすい（例: `Cadd9` `G13` `Em7♭5` `F♯/G♯`）。
- **和声リズムはモデルが決める**：コードは**1拍単位**で学習し、いつ変わるか（保持で長いコード／小節内で変化）をモデルが決める。「1小節1コード」に固定せず、コード名チップは変化ごとに区切って拍数も表示。
- **形式・モチーフもモデル任せ**：フル曲進行で学習しているため生成に**反復ループ（構造）が創発**。その反復を検出して同区間のメロディを再利用し**モチーフを再帰**（形式テンプレートは与えない。構造は毎回変わる）。
- **キー正規化・可変長**：トニック相対で学習するのでどのキーでも生成可能。曲の長さもモデル任せ（終止トークンまでの可変長）。サビ／Aメロを推定して伴奏の厚み・強弱に反映し、楽譜には強弱記号（p〜f）・クレシェンド・スラー・rit. も表示。
- **完全ブラウザ完結**：学習はオフライン（Python / PyTorch）、重みは `model.js` として同梱。ブラウザ側は依存ライブラリなしの自作推論エンジン `neural.js`（KVキャッシュつき自己回帰生成、`file://` で開くだけで動作）。

学習データは POP909（ポップスのピアノカバー）、OpenEWLD・Wikifonia（多ジャンルのリードシート）、Nottingham（民謡）、JSBコラール（Bach）、McGill Billboard・isophonics ほかのコード注釈を使用（**リポジトリには同梱していません**。取得・前処理・学習の全手順は [`docs/data-pipeline.md`](docs/data-pipeline.md) と [`tools/README.md`](tools/README.md) を参照）。

> `model.js` が無い環境でも、音楽理論ベースのルール生成に自動フォールバックします。

## コード学習（スライド形式・全52パート・4章立て）

パートを選び、**スライドを「← 前へ / 次へ →」で送りながら**学びます。パート一覧は**章見出し**（準備編／メロディ編／コード編／リズム編）でグループ表示されます。各スライドは内容に応じて次を表示します。

- **解説** … 要点を箇条書きで表示。
- **楽譜（五線譜）** … 音符・音程・スケール・コードを五線譜で表示（ト音記号・臨時記号・加線つき）。
- **鍵盤ハイライト** … 該当の音／コードを**画面下部のピアノ**に光らせて示す。
- **リズムグリッド** … リズム編では拍のグリッドを表示し、**▶ リズムを再生**で拍位置の**ドラム音**（キック／スネア／ハイハット／クラーベ／拍のクリック）を鳴らして体感。
- **概念デモ／🎵 応用例** … 音符・コードの「違い・流れ・音階・進行」は鍵盤アニメーションで再生。スケール・旋法・カデンツ・進行など「使うとどう聞こえるか」が要点の回には**応用例メロディ**（きらきら星／ii-V-I／王道・カノン進行／ブルースのリック 等）を用意。
- **✍️ 練習** … 下の鍵盤で実際に弾いて確認（スケールは1音ずつ、コードは和音、転回形は積み方まで判定）。

**スライド／練習に応じて、下部ピアノの調（左端の音）が自動で切り替わります**（例：Gメジャーのスライドでは左端が「ソ」に。鍵盤がスライドするので指の形は同じ）。

### カリキュラム（全52パート・4章立て）

```
■ 準備編（Part 1-5）
  1 さまざまな音名と音程   2 インターバルとスケール   3 さまざまなスケール
  4 五度圏と調号           5 倍音と音色（ADSR）
■ メロディ編（Part 6-13）
  6 メロディの基礎          7 傾性と調性引力
  8 ペンタトニックと民族音階  9 三つの短音階
  10 ブルーノートと特殊音階   11 教会旋法（明るい旋法）
  12 教会旋法（暗い旋法）     13 シェル理論とハモリ
■ コード編（Part 14-46）
  14 ダイアトニック①〜 転回形・機能和声・ドミナント・セカンダリ・ツーファイブ・
  パッシングディミニッシュ・裏コード・SDマイナー・クリシェ・各種テンション・UST 〜40
  41 ドミナントの本質      42 いろいろなドミナント（ドッペル／エクステンデッド）
  43 偽終止とピカルディ終止  44 特殊構造（7sus4／ハイブリッド）
  45 スペシャルファンクション・ドミナント（I7/IV7/♭VII7/♭VI7/II7/VII7）
  46 高度な応用（整列式2連続／オルタレーション／非機能配置）
■ リズム編（Part 47-52）
  47 拍と拍子      48 ビートとサブディビジョン  49 強拍とシンコペーション
  50 クラーヴェ    51 シャッフルと複合拍子     52 ポリリズムと変拍子
```

進捗（クリア済み ✓）はブラウザに自動保存されます。

## 音色（楽器）

画面上部の「音色」セレクトで楽器を切り替えられます。

- **グランドピアノ**（初期値・本物のピアノ音源 Salamander Grand Piano をサンプル再生）
- エレクトリックピアノ / オルガン / ストリングス / シンセリード / マリンバ / ベル（合成音）

> グランドピアノは初回選択時に音源ファイル（CDN）を読み込むため、少し時間がかかります。
> 読み込み状況は画面下部に表示されます。オフラインでは合成音の楽器をご利用ください。

## 使用ライブラリ・音源のクレジット

- [Tone.js](https://tonejs.github.io/)（MIT License）— Web Audio の発音・スケジューリング。CDN読み込み。
- [VexFlow](https://www.vexflow.com/)（MIT License）— 五線譜の描画。CDN読み込み。
- [Salamander Grand Piano](https://archive.org/details/SalamanderGrandPianoV3)（CC BY 3.0, Alexander Holm）— グランドピアノ音源。Tone.js ホストのサンプルをCDNから読み込み。

AI作曲モデルの学習には研究用途で公開されているデータセット（POP909, OpenEWLD, Nottingham, JSB Chorales, McGill Billboard, isophonics ほか）を使用しています。**データセット自体は本リポジトリに含まれません**。詳細と各データセットの出典は [`docs/data-pipeline.md`](docs/data-pipeline.md) を参照してください。

カリキュラムの解説テキストは本アプリのために書き下ろしたもので、内容の正確性確認に参照した資料は [`docs/curriculum.md`](docs/curriculum.md) 末尾の「出典」に記載しています（準備編・メロディ編・リズム編の構成は [SoundQuest](https://soundquest.jp/) の自由派音楽理論を参考にしています）。
